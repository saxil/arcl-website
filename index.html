arcl

Transactional coding CLI that edits files via unified diffs

A filesystem mutation engine for AI-assisted coding. Operates outside the IDE, proposes changes as explicit diffs, and never touches files without confirmation. Boring but trustworthy.

Core Features

Safe, transparent, and predictable file mutations

Transactional Edits

Add, edit, or remove files atomically. All changes shown as unified diffs before applying.

Template Scaffolding

Deterministic project creation from JSON templates. Flask, FastAPI, React, Express, and more.

Preview Mode

Use --dry-run to see exactly what would happen without applying any changes.

Automatic Rollback

Every mutation creates a .bak file. Failed patches restore original content.

Edit Guardrails

Policy-based validation via .arcl/config.json. Block dangerous patterns like eval().

Multi-Provider LLM

Gemini, OpenRouter, or local Ollama. Automatic failover when primary is unavailable.

How It Works

Simple commands, predictable results

Create Files

arcl add main.py "hello world script"

Edit Files

arcl edit main.py "add error handling"

Remove Files

arcl remove src/routes/old_route.py

Ask Questions

arcl ask src/main.py "explain the main function"

Create Projects

arcl create project "a calculator app using tkinter"

Explain Changes

arcl explain last

Security & Guardrails

Built-in protections to keep your codebase safe

Workspace Isolation

All operations are confined to a dedicated workspace directory. arcl cannot modify files outside this boundary.

WindowsC:\arcl-projects
macOS~/Desktop/arcl-projects
Linux~/arcl-projects

Forbidden Patterns

Block dangerous code patterns from ever being written. Regex-based rejection of unsafe constructs.

eval() exec() __import__()

Automatic Backups

Every file mutation creates a .bak backup. If a patch fails, the original is automatically restored.

Explicit Confirmation

No file is ever modified without your approval. All changes are shown as diffs before applying.

Edit Guardrails Configuration

Create .arcl/config.json to enforce policies:

.arcl/config.json 
{
  "allow_full_rewrites": false,
  "forbid_patterns": ["eval\\(", "exec\\(", "__import__\\("],
  "max_diff_lines": 500,
  "require_confirmation": true
}
Policy Default Description
allow_full_rewrites false Permit diffs that replace entire files
forbid_patterns ["eval\\(", ...] Regex patterns to reject in output
max_diff_lines 500 Maximum lines per diff (0 = unlimited)
require_confirmation true Always ask before applying
Fail Closed

If the LLM output is invalid, malformed, or violates policies — nothing happens. No partial writes, no corrupted files.

Installation

Clone, install, and link globally

1 Clone the repository

git clone https://github.com/saxil/arcl-cli.git

2 Install dependencies

cd arcl-cli && npm install

3 Link globally

npm link

Configuration

Set one environment variable for your LLM provider:

Gemini (default)

$env:GEMINI_API_KEY = "your-key"

OpenRouter

$env:ARCL_PROVIDER = "openrouter"
$env:OPENROUTER_API_KEY = "your-key"

Local (Ollama)

$env:ARCL_PROVIDER = "local"
$env:ARCL_LOCAL_MODEL = "codellama"

Core Principles

1

Runtime owns structure

LLM never creates directories or chooses paths

2

LLM is text-only

No tool calls, no command execution

3

All mutations explicit

User confirms every change

4

Fail closed

Invalid output = no changes
